로컬용 파이썬 프로그램 jupyterlab 설치

2026년 7월 8일 수요일

uv로 JupyterLab 실행하기 (Python 공식 CLI 설치 포함)

Python 개발 환경을 깔끔하게 유지하면서 JupyterLab을 사용하고 싶다면 uv + JupyterLab 조합이 가장 빠르고 효율적입니다. 이 문서는 Markdown 형식으로 작성되었으며, Python을 “공식 CLI 방식"으로 설치하는 방법까지 포함합니다.

1. Python 공식 CLI 설치 방법

uv는 시스템에 설치된 Python을 기반으로 동작하므로 먼저 Python을 설치해야 합니다. 아래 방법들은 모두 OS에서 공식적으로 인정하는 패키지 채널을 통한 설치 방식입니다.

macOS — Homebrew

Homebrew는 macOS에서 사실상 표준 패키지 매니저로 널리 쓰입니다.

brew install python

설치 확인:

python3 --version

Windows — Microsoft 공식 패키지 매니저 winget

winget은 Microsoft가 공식 제공하는 패키지 매니저이며, Python.org 공식 배포판을 그대로 설치합니다.

Python.Python 이라는 단독 패키지 ID는 존재하지 않습니다. Python.Python.3.13처럼 버전이 명시된 ID만 winget 저장소에 등록되어 있으므로, 먼저 검색으로 현재 사용 가능한 ID를 확인한 뒤 설치하세요.

winget search Python.Python
winget install --id Python.Python.3.13 -e

설치 확인:

python --version

Ubuntu / Debian — 공식 apt 저장소

리눅스 배포판의 공식 저장소는 Python.org와 연동되어 있어 공식 설치로 인정됩니다.

  • 다만 최신 Ubuntu에는 Python3가 기본 설치되어 있는 경우가 많으니, 먼저 버전을 확인해보는 것도 좋습니다.
python3 --version   # 이미 설치돼 있는지 먼저 확인

sudo apt update
sudo apt install python3 python3-venv python3-pip

설치 확인:

python3 --version

Fedora / RHEL — 공식 dnf 

sudo dnf install python3

설치 확인:

python3 --version

2. uv 설치

uv는 초고속 패키지/환경 관리 도구입니다.

설치 

curl -LsSf https://astral.sh/uv/install.sh | sh

설치 확인 

uv --version

3. 프로젝트 초기화

프로젝트 폴더를 만들고 uv 환경을 초기화합니다.

mkdir myproject
cd myproject
uv init

uv init 실행 시 생성되는 파일

  • pyproject.toml — 프로젝트 설정 및 패키지 관리 파일
  • .python-version — 프로젝트에서 사용할 Python 버전 지정
  • README.md, main.py 등 기본 파일

이 시점에는 아직 .venv/ 가상환경이 생성되지 않습니다. .venv/와 uv.lock은 uv add, uv run, uv sync 중 하나를 처음 실행할 때 자동으로 생성됩니다.

4. JupyterLab 설치

프로젝트 환경에 JupyterLab을 추가합니다.

uv add jupyterlab

이 명령은 다음을 수행합니다:

  • 이 시점에 .venv/ 가상환경이 아직 없다면 자동으로 생성
  • pyproject.toml에 jupyterlab 의존성 추가
  • .venv 환경에 JupyterLab 설치

5. JupyterLab 실행

설치가 완료되면 아래 명령으로 바로 실행할 수 있습니다.

uv run jupyter lab

실행 시 uv가 자동으로 처리하는 것

  • .venv 가상환경 자동 활성화
  • 해당 환경의 JupyterLab 실행 파일 호출
  • 별도의 커널 등록(ipykernel install) 없이 .venv의 Python 커널이 그대로 연결됨
  • 브라우저에서 JupyterLab 자동 오픈

브라우저에서 다음 주소가 열립니다:

http://localhost:8888

6. 자주 사용하는 uv 명령어

패키지 목록 확인 

uv pip list

패키지 제거 

uv remove 패키지명

uv 환경 삭제 

rm -rf .venv

7. 전체 흐름 요약 

# Python 설치 (OS별 공식 CLI 방식 참고)

# uv 설치
curl -LsSf https://astral.sh/uv/install.sh | sh

# 프로젝트 생성 및 초기화
mkdir myproject
cd myproject
uv init

# JupyterLab 설치 (이 단계에서 .venv가 생성됨)
uv add jupyterlab

# 실행
uv run jupyter lab

마무리

uv는 기존 pip/venv보다 훨씬 빠르고 직관적이며, 프로젝트별 환경 관리가 매우 깔끔합니다. 특히 JupyterLab을 사용할 때 uv run만으로 즉시 실행할 수 있다는 점이 큰 장점입니다.

로컬로 파이썬을 하고 싶다보니 이렇게 정리하게 되었습니다.
