긴급 정리! 분류 알고리즘 평가 지표

2026년 7월 9일 목요일

들어가며

이번에는 모델들 중에 긴급히 정리할 내용이 있어서 정리해보려고 합니다.

데이터분석에는 다양한 모델이 있는데, 크게 예측, 분류, 군집, 차원축소 4가지로 나뉩니다.

그중 분류 모델에서 단순히 “정확도"만으로는 모델의 진짜 성능을 알기 어려운 이유와, 이를 보완하는 평가 지표(정밀도, 재현율, F1 Score, ROC-AUC 등)를 정리하겠습니다.

1. 혼동 행렬 (Confusion Matrix)

분류 모델의 성능을 평가하기 위해 사용되는 행렬

모델이 예측한 값과 실제 데이터를 교차해서 4가지 경우로 나눕니다.

예측: Positive 예측: Negative
실제: Positive True Positive (TP) False Negative (FN) — Type-2 error
실제: Negative False Positive (FP) — Type-1 error True Negative (TN)
  • Positive (긍정 클래스, 양성): 모델이 관심 있는 클래스
  • Negative (부정 클래스, 음성): 모델이 관심 없는 클래스
  • 예시: 스팸 메일 분류기의 경우, 스팸 메일 = Positive, 비 스팸 메일 = Negative

이 4가지 값(TP, FN, FP, TN)을 조합해서 아래와 같은 다양한 평가 지표를 계산할 수 있습니다.

지표 공식
Recall (재현율) TP / (TP + FN)
Specificity TN / (TN + FP)
Precision (정밀도) TP / (TP + FP)
Negative Predictive Value TN / (TN + FN)
Accuracy (정확도) (TP + TN) / (TP + TN + FP + FN)

2. 정확도 (Accuracy)

전체 데이터 중에서 제대로 분류된 데이터의 비율

$$ Accuracy = \frac{TP + TN}{TP + TN + FP + FN} $$

  • 모델이 얼마나 정확하게 분류하는지를 나타냄
  • 일반적으로 분류 모델의 주요 평가 방법으로 사용
  • 클래스 비율이 불균형할 경우, 평가 지표의 신뢰성을 잃음

불균형한 클래스에서의 정확도 문제

전체 100만 개의 항공 데이터 중,

  • 90만 개가 정상 운행 (Negative)
  • 10만 개만 지연 운행 (Positive)

인 상황에서, 모델이 전체 결과를 모두 “지연되지 않았다"고 예측해버려도

$$ 정확도 = \frac{0개 + 90만개}{100만개} \times 100 = 90% $$

로 계산됩니다.

정확도만 보면 90%로 꽤 높아 보이지만, 실제로는 지연 운행(관심 클래스)을 단 하나도 맞히지 못한 무의미한 모델입니다.

이처럼 클래스가 불균형할 때는 정확도만으로 모델을 판단하면 위험하며, 다양한 평가지표를 함께 고려해야 합니다.

3. 정밀도 (Precision)

모델이 Positive라고 분류한 데이터 중 실제로 Positive인 데이터의 비율

$$ Precision = \frac{TP}{TP + FP} $$

  • 모델이 예측한 긍정 사례가 실제로 맞는 비율 (= 예측의 정확도)
  • 일반적으로 정밀도는 Type-1 Error를 줄이는 것이 중요한 상황에서 주로 사용
    • Type-1 Error: 실제로는 Negative인데 Positive로 예측한 사례

예시: 스팸 메일 판정 문제

정상 메일(Negative)이 스팸(Positive)으로 잘못 분류(False Positive)될 경우, 중요한 메일을 전달받지 못하는 심각한 상황이 발생합니다.

이런 경우에는 정밀도를 높여 False Positive를 최소화하는 것이 중요합니다.

4. 재현율 (Recall)

실제로 Positive인 데이터 중 모델이 Positive로 분류한 데이터의 비율

$$ Recall = \frac{TP}{TP + FN} $$

  • 모델이 실제 긍정 사례를 놓치지 않는 비율 (= 관심 사례 포착 정도)
  • 일반적으로 재현율은 Type-2 Error를 줄이는 것이 중요한 상황에서 주로 사용
    • Type-2 Error: 실제로는 Positive인데 Negative로 예측한 사례

예시: 질병 진단 문제

암 환자(Positive)를 놓치는 경우(False Negative)는 큰 위험을 초래합니다.

즉, 모델이 실제 환자를 놓치지 않도록 보장해야 하며, Recall을 높이면 실제 암 환자를 더 많이 포착하게 되어 놓치는 환자를 최소화할 수 있습니다.

정밀도 vs 재현율: 스팸 필터는 정밀도가 중요하고(정상 메일을 스팸으로 오분류하면 안 됨), 암 진단은 재현율이 중요합니다(환자를 놓치면 안 됨). 이처럼 문제의 특성에 따라 어떤 지표를 우선할지 달라집니다.

5. F1 Score

정밀도와 재현율의 조화 평균

$$ F1 = \frac{2 \times Precision \times Recall}{Precision + Recall} $$

  • 모델의 정확성과 완전성을 동시에 고려하여 모델의 성능을 평가
  • 데이터 불균형이 심각할 때 정밀도와 재현율을 절충해서 보고 싶을 때 사용

6. FPR (False Positive Rate)

실제로 Negative인 데이터 중 모델이 Positive로 분류한 데이터의 비율

$$ FPR = \frac{FP}{FP + TN} $$

예시: 게임 내 비정상 사용자 검출

FPR이 높다는 것은 정상 사용자를 비정상 사용자로 잘못 검출하는 경우가 많다는 의미입니다.

이 상태에서 비정상 사용자에게 페널티(제재)를 부여하면, 선의의 정상 사용자가 피해를 입게 됩니다.

7. ROC Curve와 AUC

  • x축: False Positive Rate, y축: Recall(True Positive Rate) 값으로 시각화한 그래프
  • 커브가 왼쪽 위로 당겨질수록 Classifier의 성능이 향상됨을 의미
  • AUC (Area Under Curve) = ROC Curve 아래의 면적
    • AUC = 0.5인 대각선 부분은 Classifier의 최하 성능(랜덤 추측 수준)을 의미
    • AUC가 1에 가까울수록 좋은 분류기

8. 코드로 평가 지표 구현하기

강의에서 다룬 sklearn.metrics를 활용한 평가 지표 계산 코드입니다.

아래 코드는 model이 이미 학습(fit)까지 완료된 분류 모델이고, X_valid, y_valid는 검증용 데이터라고 가정합니다.

from sklearn.metrics import accuracy_score, precision_score, recall_score, f1_score

# 검증 데이터에 대한 예측
y_pred = model.predict(X_valid)

# 정확도 계산
accuracy = accuracy_score(y_valid, y_pred)

# 정밀도 계산
precision = precision_score(y_valid, y_pred)

# 재현율 계산
recall = recall_score(y_valid, y_pred)

# F1 Score 계산
f1 = f1_score(y_valid, y_pred)

print("Accuracy:", accuracy)
print("Precision:", precision)
print("Recall:", recall)
print("F1 Score:", f1)

추가 예시 ① — 혼동 행렬 직접 확인하기 

from sklearn.metrics import confusion_matrix, ConfusionMatrixDisplay
import matplotlib.pyplot as plt

cm = confusion_matrix(y_valid, y_pred)
print("혼동 행렬:\n", cm)

# 시각화
disp = ConfusionMatrixDisplay(confusion_matrix=cm, display_labels=["Negative", "Positive"])
disp.plot(cmap="Blues")
plt.show()

추가 예시 ② — 불균형 데이터에서 정확도의 함정 재현하기

바로 위 2번 섹션에서 다룬 “항공 데이터 90%” 예시를, 실제 코드로 재현해서 정확도만 보면 안 되는 이유를 직접 확인해봅니다.

import numpy as np
from sklearn.metrics import accuracy_score, precision_score, recall_score, f1_score

# 항공 데이터 예시: 90만개 정상(0), 10만개 지연(1)
y_true = np.array([0] * 900_000 + [1] * 100_000)

# 모델이 항상 "정상"이라고만 예측하는 극단적인 경우
y_pred_dummy = np.zeros(1_000_000)

print("Accuracy:", accuracy_score(y_true, y_pred_dummy))   # 0.9 (90%)
print("Precision:", precision_score(y_true, y_pred_dummy, zero_division=0))  # 0.0
print("Recall:", recall_score(y_true, y_pred_dummy, zero_division=0))        # 0.0
print("F1 Score:", f1_score(y_true, y_pred_dummy, zero_division=0))          # 0.0

정확도는 90%로 높지만, Precision/Recall/F1은 모두 0에 가까워 이 모델이 사실상 아무것도 예측하지 못하고 있음을 명확히 보여줍니다.

추가 예시 ③ — ROC Curve와 AUC 그리기 

from sklearn.metrics import roc_curve, roc_auc_score
import matplotlib.pyplot as plt

# 모델이 예측한 Positive 클래스 확률
y_proba = model.predict_proba(X_valid)[:, 1]

fpr, tpr, thresholds = roc_curve(y_valid, y_proba)
auc = roc_auc_score(y_valid, y_proba)

plt.plot(fpr, tpr, label=f"ROC Curve (AUC = {auc:.2f})")
plt.plot([0, 1], [0, 1], linestyle="--", color="gray", label="Random (AUC = 0.5)")
plt.xlabel("False Positive Rate")
plt.ylabel("True Positive Rate (Recall)")
plt.title("ROC Curve")
plt.legend()
plt.show()

추가 예시 ④ — classification_report로 한 번에 확인하기 

from sklearn.metrics import classification_report

# Precision, Recall, F1 Score를 클래스별로 한 번에 확인
print(classification_report(y_valid, y_pred, target_names=["Negative", "Positive"]))

마무리

오늘 정리한 분류 평가 지표를 한 줄로 요약하면 다음과 같습니다.

지표 핵심 질문 중요하게 쓰이는 상황
Accuracy 전체적으로 얼마나 맞혔나? 클래스가 균형 잡혀 있을 때
Precision Positive로 예측한 것 중 진짜는 몇 개? 오탐(FP)의 비용이 클 때 (예: 스팸 필터)
Recall 진짜 Positive 중 몇 개를 찾아냈나? 미탐(FN)의 비용이 클 때 (예: 질병 진단)
F1 Score Precision과 Recall의 균형은? 데이터가 불균형할 때 종합 평가
FPR / ROC-AUC Negative를 얼마나 Positive로 착각했나? 임계값(threshold)에 따른 전체 성능 비교

정확도 하나만 보고 모델을 평가하는 것은 위험하며, 문제 상황에 맞는 지표를 선택하는 것이 중요합니다.
