빅데이터의 이론개념

2026년 7월 5일 일요일

들어가며

오늘은 간단한 이론개념을 간단하게 정리하려고 합니다.

1. AI와 빅데이터

인공지능 > 머신러닝 > 딥러닝

인공지능 ⊃ 머신러닝 ⊃ 딥러닝 세 개념은 포함 관계입니다.

인공지능 (Artificial Intelligence)

  • 컴퓨터 시스템이 인간과 유사한 지능을 가지도록 하는 기술
  • 학습, 추론, 인식, 계획, 지각, 개체 조작 능력 문제 등을 다룸

머신러닝 (Machine Learning)

  • 인공지능의 한 분야
  • 데이터를 통해 컴퓨터가 스스로 학습하고 의사결정을 내리는 능력을 갖게 함
  • 지도학습(Supervised Learning): 정답(label)이 있는 데이터로 학습
    • 예 : 강아지/고양이 사진에 라벨을 붙여 분류 모델 학습
  • 비지도학습(Unsupervised Learning): 정답이 없는 데이터로 학습
    • 예 : 라벨 없는 고객 데이터를 그룹으로 클러스터링

딥러닝 (Deep Learning)

  • 사람의 신경망을 모사한 인공 신경망 기반의 머신러닝 한 종류
  • 기존에 풀기 어려웠던 복잡한 문제(이미지 인식, 자연어 처리 등)를 해결 가능

빅데이터 (Big Data)

  • 통상적으로 사용되는 데이터 수집·관리·처리 소프트웨어의 수용 한계를 넘어서는 크기의 데이터

  • 데이터 양이 방대하고 지속적으로 증가하여 전통적인 방법으로는 분석할 수 없는 데이터

    • 예 : 하루에도 수십억 건씩 쌓이는 SNS 게시글, 실시간 카드 결제 로그, IoT 센서 데이터 등

2. 빅데이터의 특징

빅데이터 3V → 4V → 5V로 확장

구분의미설명
Volume (규모)양테라바이트 수준 이상의 규모
Variety (다양성)형태정형/비정형의 다양한 형태
Velocity (속도)속도실시간으로 생산·처리·유통되는 빠른 데이터
Value (가치)*가치함축된 가치를 찾아내는 것이 중요
Veracity (진실성)*신뢰성데이터의 신뢰성과 타당성 확보 필요

데이터의 종류

종류설명예시
정형 데이터 (Structured)열과 행으로 이루어진 테이블 형식의 구조화된 데이터엑셀 표, RDB 테이블
반정형 데이터 (Semi-structured)정형은 아니지만 어느 정도 구조를 가진 데이터XML, HTML, JSON
비정형 데이터 (Unstructured)특정 구조를 갖지 않는 데이터비디오, 오디오, 텍스트, 이미지

3. 데이터 분석 계획

데이터 분석이란?

  • 특정 타깃의 패턴이나 트렌드 같은 유용한 인사이트를 발견하기 위해 데이터를 수집 → 정리 → 조작 → 분석하는 전 과정
  • 통계 분석, 머신러닝, 데이터 마이닝 등의 방법을 사용

분석 방법론 두 가지

  • 확증적 데이터 분석 (CDA, Confirmatory Data Analysis) 가설 설정 → 데이터 수집 → 통계 분석 → 가설 검증 (반복)

    • 예 : “광고 노출이 많을수록 매출이 늘어난다"는 가설을 세우고, 데이터를 통해 검증

  • 탐색적 데이터 분석 (EDA, Exploratory Data Analysis) 데이터 수집 → 시각화 탐색 → 패턴 도출 → 인사이트 발견 (반복)

    • 예 : 뚜렷한 가설 없이 판매 데이터를 시각화하며 숨겨진 패턴을 찾아냄

데이터 분석의 장점

  • 효율적인 의사 결정에 도움
  • 타깃 이해도와 만족도를 높임
  • 마케팅 최적화를 도움
  • 업무 효율성을 높임
  • 예측 가능성을 높여 비즈니스 리스크 관리를 강화함

데이터 분석 순서 (5단계)

  1. 문제 정의: 가치 있는 문제 탐색 후 분석 대상과 목적을 객관적·구체적으로 설정
  2. 데이터 수집: 크롤링, Open API 등을 활용해 필요한 데이터 확보
  3. 데이터 전처리: 수집된 raw data의 이상치·중복값·결측값 처리
  4. 데이터 모델링: 통계 기반, 머신러닝, 딥러닝 등으로 목적에 맞는 모델 생성
  5. 결과 분석 및 시각화: 결과를 시각화하고 분석하여 인사이트 도출

4. 빅데이터 활용 기술

기법설명예시
연관 규칙 학습(Association rule learning)변수 간의 관계를 발견하는 기법슈퍼마켓에서 (감자, 양파)를 산 사람이 햄버거용 재료도 같이 사더라
유형(분류) 분석(Classification tree analysis)통계적 분류로 사건이 속할 범주를 찾는 기법소득·가족 수·교육 연수에 따라 “구매자/비구매자” 분류
유전 알고리즘(Genetic algorithms)자연선택·돌연변이 메커니즘으로 최적해를 점진적으로 찾는 기법응급실 의사 배치 최적화, 방송 편성 시간표 최적화
기계 학습(Machine learning)훈련 데이터로 학습한 특성을 활용해 예측하는 기법넷플릭스 영화 추천, 유튜브 알고리즘
회귀 분석(Regression analysis)독립변수 변화에 따른 종속변수 변화를 통해 두 변인의 관계를 파악나이와 구매 차량 타입의 관계, 만족도와 충성도의 관계
감정 분석(Sentiment analysis)자연어 데이터에서 감정 시그널을 추출하는 기법리뷰가 긍정/중립/부정 중 어디에 속하는지 분류
소셜 네트워크 분석(Social network analysis)사회 관계망을 분석해 영향력 있는 사람이나 관계를 파악인플루언서(에너자이저/브로커/커넥터) 파악

5. 데이터 변환

이상치 (Outlier)

  • 관측되는 데이터의 범위에서 크게 벗어나는 값

  • 결과적으로 잘못된 분석 결과를 초래할 수 있음

  • 예 : 데이터 4, 4, 5, 5, 5, 5, 6, 6, 6, 7, 7 (평균 5.45, 표준편차 1.04) 여기에 이상치 300이 추가되면 평균 30.00, 표준편차 85.03로 완전히 왜곡됨 적절한 이상치 처리가 필수적!

이상치 발견 방법

  1. Box-plot: Q1(25%)-1.5×IQR ~ Q3(75%)+1.5×IQR 범위를 벗어나면 이상치
  2. Histogram: 분포에서 멀리 떨어진 낮은 빈도 구간을 시각적으로 확인
  3. Scatter Plot: 산점도 상에서 전체 추세를 벗어나는 점을 확인

이상치 처리 방법

  1. 제거: 이상치 데이터를 아예 삭제
  2. 대체: 평균값/중앙값 또는 정해진 최댓값·최솟값으로 대체

결측치 (Missing Value)

  • 값이 누락된 데이터 (NA, NaN 등으로 표시)

결측치 처리 방법

  1. 제거: 결측치가 있는 행을 삭제
  2. 대체: 전체 평균(Global Mean) 또는 그룹별 평균(Group Mean) 등으로 대체
  • 예 : OS가 Android인 행의 결측 매출을 “Android 평균 매출"로 채우는 방식이, 전체 평균으로 채우는 것보다 더 정확할 수 있음
